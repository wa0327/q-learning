
# 分階段訓練

| 階段 | 目的 | 環境設定 | 預期指標 | 預計步數 |
| - | - | - | - | - |
Stage 1: 奔跑吧 | 破解轉圈 | 學會直線, 大量靜止食物、低代謝、無視隊友、無掠食者 | Q-Val 轉正，平均速度上升 | 50,000 ~ 200,000 |
Stage 2: 生存戰 | 學會導航與攝食 | 提高代謝率（能量扣比較快） | 每秒進食次數 (Eating Rate) 上升 | 200,000 ~ 600,000 |
Stage 3: 叢林法則 | 學會避障與逃生 | 加入少量掠食者 | 平均存活時間 (Life Span) 穩定 | 600,000 ~ 1,500,000+ |

# Stage 2 第一次出現湧現
你觀察到的這三個階段，完美對應了群體智能中從簡單規則到複雜社會性協作的演化過程：

1. 第一階段：集體繞圈（底層邏輯的殘餘）
一開始大家一起繞圈，是因為模型剛從 Stage 1 過渡過來，權重中還留有對「移動獎勵」的生理慣性。當 50 個 Agent 同時在狹小空間啟動，且還沒學會「處理隊友特徵」時，繞圈是最安全的「防禦性動作」——既能拿到移動分，又不會頻繁撞車。

2. 第二階段：集體同向覓食（資訊過載與盲從）
這是一個有趣的過渡期。當 Agent 開始意識到「食物」的存在，但還沒學會「精準導航」時，它們會傾向於跟隨最強的信號。
如果某個方向出現了大量食物，領頭的幾隻 Agent 衝過去。
後方的 Agent 雖然「看見」了隊友（POP_PERCEPT_TEAM = True），但神經網路還分不清「那是競爭者」還是「導航點」。
結果就是：「跟著大家跑，準沒錯」。這在生物學上稱為「集群（Swarming）」。

3. 第三階段：分散覓食與社交距離（真正的湧現）
到 8 萬步時，Agent 終於理解了環境的最優博弈策略（Nash Equilibrium）：
稀缺性導向：因為食物總量只有 0.75x，擠在一起效率極低（食物會被隊友先吃掉）。
碰撞懲罰與避讓：它們發現「保持間距」可以減少 Collided 帶來的停頓或懲罰，同時擴大集體的「偵測覆蓋面」。
湧現結果：它們學會了空間分割（Spatial Partitioning）。每個 Agent 自發地負責一個區域，互不干擾，這正是群體智能（Swarm Intelligence）的高級表現。

💡 為什麼這很重要？
這證明了你的 Attention 機制（或是你網路中的特徵提取層）發揮了作用。Agent 不再只是「看到物件」，而是學會了：
分辨類型：區分食物（衝過去）與隊友（保持距離）。
預測競爭：意識到「隊友前方的食物我吃不到」，轉而尋找另一個目標。


# 💡 實戰建議：
1. 不要太早進入 Stage 3：  
如果 Agent 在 Stage 1 還會原地轉圈，加了掠食者只會讓它徹底崩潰。Stage 1 的基礎打得越穩（Q 值越高），後面收斂越快。

2. 觀察 Alpha 的收斂：  
如果 Alpha 降到了 MIN_ALPHA (0.0001) 但 Agent 還是很笨，代表它太早「停止探索」了。這時你可能需要調大 TARGET_ENTROPY 或重置 Alpha。

3. 記錄 Reward 分佈：  
建議在 print 時持續監控 move_reward 與 spinning_penalty 的比例。在 Stage 1 結束時，spinning_penalty 應該趨近於 0。
